# Brand Voice

Apply the Forever Fireflies brand voice when writing any user-facing copy.

TRIGGER when: writing or editing UI strings, push notification text, onboarding copy, empty state messages, error messages, app store descriptions, email templates, social media copy, or any other text that users or customers will read.

DO NOT TRIGGER when: refactoring code, writing backend logic, database migrations, infrastructure changes, or any task with no user-facing text.

## Instructions

1. Read `/BRAND_VOICE.md` — it is the single source of truth for tone, word choice, banned words, emoji rules, and channel-specific guidelines.
2. For in-app copy specifically, also reference the Tone & Language table in `docs/design/design-style.md` (lines 519-543) — that table governs in-app writing and the brand voice doc defers to it.
3. Apply the three voice traits: **Quiet** (minimal words, big feeling), **Warm** (present, gentle, like a friend), **Real** (honest, grounded, never performative).
4. Check all copy against the Banned Words list before finalizing.
5. Follow the channel-specific format rules for the relevant context (push notifications, emails, social, in-app).
6. When in doubt, write toward the 10 Reference Copy examples at the bottom of `BRAND_VOICE.md` — they are the gold standard.
